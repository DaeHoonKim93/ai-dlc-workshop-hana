## Ambiguity 1: 고객 접근 방식과 기존 요구사항 간 차이

원래 요구사항 문서(table-order-requirements.md)에서는 **"테이블 태블릿 자동 로그인 및 세션 관리"** 기능이 정의되어 있습니다 (관리자가 태블릿에 매장ID/테이블번호/비밀번호를 1회 설정하면 자동 로그인).

그런데 Q10에서 **"B) QR코드 스캔으로 접속하는 모바일 웹"**을 선택하셨습니다.

이 경우 고객 인증/세션 방식이 달라집니다:
- **태블릿 방식**: 관리자가 사전 설정 → 자동 로그인 → 고정 디바이스
- **QR코드 방식**: 고객이 자신의 스마트폰으로 QR 스캔 → 테이블 식별 → 별도 로그인 불필요

### Clarification Question 1
고객 접근 방식을 어떻게 확정하시겠습니까?

A) QR코드 전용 — 각 테이블에 QR코드를 비치하고, 고객이 자신의 스마트폰으로 스캔하여 주문 (태블릿 자동 로그인 기능 제거, 테이블 비밀번호 불필요)
B) 태블릿 전용 — 원래 요구사항대로 태블릿에 자동 로그인 설정 (QR코드 불필요)
C) 둘 다 지원 — 태블릿 자동 로그인 + QR코드 모바일 웹 모두 지원
D) Other (please describe after [Answer]: tag below)

[Answer]: B
