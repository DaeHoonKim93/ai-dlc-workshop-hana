# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

| 항목 | 내용 |
|------|------|
| **사용자 요청** | 테이블오더 서비스 구축 (디지털 주문 시스템) |
| **요청 유형** | New Project (신규 프로젝트) |
| **범위 추정** | System-wide (전체 시스템 구축) |
| **복잡도 추정** | Complex (다수의 컴포넌트, 실시간 통신, 인증, 대규모 처리량) |
| **프로젝트 유형** | Greenfield |

## 1. 기술 스택 결정사항

| 영역 | 기술 | 비고 |
|------|------|------|
| **Backend** | Java + Spring Boot | REST API 서버 |
| **Frontend** | React (TypeScript) | 고객용 + 관리자용 SPA |
| **Database** | PostgreSQL | 관계형 데이터베이스 |
| **Cloud** | AWS | 프로덕션 배포 환경 |
| **이미지 저장소** | AWS S3 | 메뉴 이미지 업로드/관리 |
| **실시간 통신** | SSE (Server-Sent Events) | 주문 실시간 모니터링 |

## 2. 기능 요구사항 (Functional Requirements)

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리

**설명**: 매장 내 각 테이블에 설치된 태블릿에서 고객이 별도 로그인 없이 즉시 주문할 수 있도록 자동 인증 제공

**세부 요구사항**:
- FR-01-01: 관리자가 태블릿 초기 설정 시 매장 식별자, 테이블 번호, 테이블 비밀번호를 입력
- FR-01-02: 초기 설정 정보를 로컬 스토리지에 저장
- FR-01-03: 1회 로그인 성공 후 저장된 정보로 자동 로그인 수행
- FR-01-04: 태블릿 세션은 16시간 유지
- FR-01-05: 세션 만료 시 저장된 정보로 자동 재로그인 시도

### FR-02: 메뉴 조회 및 탐색

**설명**: 고객이 매장의 메뉴를 쉽게 탐색하고 선택할 수 있도록 지원

**세부 요구사항**:
- FR-02-01: 메뉴 화면이 기본 화면으로 항상 표시
- FR-02-02: 2단계 카테고리별 메뉴 분류 및 표시 (대분류 > 소분류)
- FR-02-03: 메뉴 상세 정보 표시 (메뉴명, 가격, 설명, 이미지)
- FR-02-04: 카테고리 간 빠른 이동 지원
- FR-02-05: 카드 형태의 메뉴 레이아웃
- FR-02-06: 터치 친화적 버튼 크기 (최소 44x44px)

### FR-03: 장바구니 관리

**설명**: 주문 전 선택한 메뉴를 임시 저장하고 수정할 수 있는 기능

**세부 요구사항**:
- FR-03-01: 메뉴 추가/삭제
- FR-03-02: 수량 조절 (증가/감소)
- FR-03-03: 총 금액 실시간 계산
- FR-03-04: 장바구니 비우기
- FR-03-05: 클라이언트 측 로컬 저장 (페이지 새로고침 시에도 유지)
- FR-03-06: 서버 전송은 주문 확정 시에만 수행

### FR-04: 주문 생성

**설명**: 장바구니의 메뉴를 실제 주문으로 전환

**세부 요구사항**:
- FR-04-01: 주문 내역 최종 확인 화면 표시
- FR-04-02: 주문 확정 버튼
- FR-04-03: 주문 성공 시 주문 번호 표시 (5초간)
- FR-04-04: 주문 성공 시 장바구니 자동 비우기
- FR-04-05: 주문 성공 시 메뉴 화면으로 자동 리다이렉트
- FR-04-06: 주문 실패 시 에러 메시지 표시 및 장바구니 유지
- FR-04-07: 주문 정보에 매장 식별 정보, 테이블 식별 정보, 주문 메뉴 목록(메뉴명, 수량, 단가), 총 주문 금액, 세션 ID 포함

### FR-05: 주문 내역 조회

**설명**: 현재 테이블의 주문 이력 확인

**세부 요구사항**:
- FR-05-01: 주문 시간 순 정렬
- FR-05-02: 주문별 상세 정보 표시 (주문 번호, 주문 시각, 주문 메뉴 및 수량, 주문 금액, 주문 상태)
- FR-05-03: 주문 상태 4단계 표시 (대기중/접수/준비중/완료)
- FR-05-04: 현재 테이블 세션 주문만 표시 (이전 식사 세션 제외)
- FR-05-05: 매장 이용 완료 처리된 주문은 제외
- FR-05-06: 페이지네이션 또는 무한 스크롤

### FR-06: 관리자 인증

**설명**: 관리자가 매장 관리 시스템에 접근하기 위한 인증

**세부 요구사항**:
- FR-06-01: 매장 식별자, 사용자명, 비밀번호 입력
- FR-06-02: 16시간 세션 유지
- FR-06-03: JWT 토큰 기반 인증
- FR-06-04: 브라우저 새로고침 시 세션 유지
- FR-06-05: 16시간 후 자동 로그아웃
- FR-06-06: 비밀번호 bcrypt 해싱
- FR-06-07: 로그인 시도 제한 (brute-force 방지)
- FR-06-08: 역할별 권한 구분 (매니저, 직원 등)

### FR-07: 실시간 주문 모니터링

**설명**: 들어오는 주문을 실시간으로 확인하고 관리

**세부 요구사항**:
- FR-07-01: SSE 기반 주문 목록 실시간 업데이트
- FR-07-02: 그리드/대시보드 레이아웃 (테이블별 카드 형태)
- FR-07-03: 각 테이블 카드에 총 주문액 표시
- FR-07-04: 최신 주문 n개 미리보기
- FR-07-05: 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- FR-07-06: 주문 상태 변경 (대기중 → 접수 → 준비중 → 완료)
- FR-07-07: 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- FR-07-08: 2초 이내 주문 표시
- FR-07-09: 테이블별 필터링 기능

### FR-08: 테이블 관리

**설명**: 테이블별 주문 상태 관리 및 세션 라이프사이클 관리

**세부 요구사항**:
- FR-08-01: 테이블 태블릿 초기 설정 (테이블 번호, 비밀번호 설정, 16시간 세션 생성)
- FR-08-02: 특정 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
- FR-08-03: 테이블 세션 시작 (첫 주문 시) 및 종료 관리 (이용 완료)
- FR-08-04: 이용 완료 시 해당 세션 주문 내역을 과거 이력으로 이동
- FR-08-05: 이용 완료 시 테이블 현재 주문 목록 및 총 주문액 0으로 리셋
- FR-08-06: 과거 주문 내역 조회 (시간 역순, 날짜 필터링)
- FR-08-07: 성공/실패 피드백 표시

### FR-09: 메뉴 관리

**설명**: 메뉴 정보를 동적으로 관리

**세부 요구사항**:
- FR-09-01: 메뉴 조회 (카테고리별)
- FR-09-02: 메뉴 등록 (메뉴명, 가격, 설명, 카테고리, 이미지 URL)
- FR-09-03: 메뉴 수정
- FR-09-04: 메뉴 삭제
- FR-09-05: 메뉴 노출 순서 조정
- FR-09-06: 이미지 S3 업로드 지원
- FR-09-07: 필수 필드 검증
- FR-09-08: 가격 범위 검증

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능

- NFR-01-01: 피크 시간 기준 분당 500건 이상 동시 주문 처리
- NFR-01-02: 매장당 테이블 100개 이상 지원
- NFR-01-03: SSE를 통한 주문 알림 2초 이내 전달
- NFR-01-04: API 응답 시간 평균 200ms 이내
- NFR-01-05: 페이지 로딩 시간 3초 이내

### NFR-02: 보안

- NFR-02-01: JWT 토큰 기반 인증/인가
- NFR-02-02: 비밀번호 bcrypt 해싱
- NFR-02-03: HTTPS 통신 필수
- NFR-02-04: 로그인 시도 제한 (brute-force 방지)
- NFR-02-05: CORS 정책 적용
- NFR-02-06: SQL Injection 방지 (parameterized queries)
- NFR-02-07: XSS 방지 (입력값 검증/이스케이프)
- NFR-02-08: Security Extension Rules 전체 적용 (SECURITY-01 ~ SECURITY-15)

### NFR-03: 가용성

- NFR-03-01: AWS 클라우드 기반 배포
- NFR-03-02: 서비스 가용성 99.9% 목표

### NFR-04: 확장성

- NFR-04-01: 수평 확장 가능한 아키텍처
- NFR-04-02: 데이터베이스 커넥션 풀링

### NFR-05: 사용성

- NFR-05-01: 터치 친화적 UI (최소 44x44px 터치 타겟)
- NFR-05-02: 직관적인 네비게이션
- NFR-05-03: 반응형 디자인 (태블릿 최적화)
- NFR-05-04: 접근성 준수 (WCAG 기본 수준)

### NFR-06: 유지보수성

- NFR-06-01: 계층형 아키텍처 (Controller - Service - Repository)
- NFR-06-02: 코드 문서화 (Javadoc, JSDoc)
- NFR-06-03: 단위 테스트 커버리지

## 4. 비즈니스 컨텍스트

### 매장 구조
- **단일 매장 전용**: 하나의 배포 인스턴스 = 하나의 매장
- **테이블 규모**: 100개 이상
- **관리자 구조**: 다수의 관리자 계정, 역할별 권한 구분 (매니저, 직원 등)

### 사용자 유형
| 사용자 | 접근 방식 | 주요 활동 |
|--------|-----------|-----------|
| **고객** | 태블릿 (자동 로그인) | 메뉴 조회, 장바구니, 주문, 주문 내역 확인 |
| **매니저** | 브라우저 (ID/PW 로그인) | 전체 관리 기능 (주문 모니터링, 테이블 관리, 메뉴 관리, 직원 관리) |
| **직원** | 브라우저 (ID/PW 로그인) | 주문 모니터링, 주문 상태 변경 (제한된 권한) |

### 주문 상태 흐름
```
대기중 → 접수 → 준비중 → 완료
```

### 테이블 세션 라이프사이클
```
테이블 비어있음 → 첫 주문 (세션 시작) → 추가 주문들 → 이용 완료 (세션 종료) → 과거 이력 이동 → 테이블 리셋
```

## 5. 제외 범위 (Out of Scope)

다음 기능은 이번 MVP에서 구현하지 않습니다:

- 결제 처리 (카드, 현금, 디지털 지갑, PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA, OTP)
- 파일/컨텐츠 관리 (이미지 리사이징/최적화, CMS, 광고)
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 재고 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도 API, 번역 API)

## 6. 메뉴 카테고리 구조

2단계 카테고리 구조:

```
대분류 (Category)
  └── 소분류 (SubCategory)
        └── 메뉴 아이템 (MenuItem)

예시:
음료 (Category)
  ├── 커피 (SubCategory)
  │     ├── 아메리카노
  │     └── 카페라떼
  └── 주스 (SubCategory)
        ├── 오렌지주스
        └── 사과주스
```

## 7. 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보
- **Table**: 테이블 정보 (매장 소속)
- **TableSession**: 테이블 세션 (시작~종료)
- **Admin/User**: 관리자 계정 (역할 포함)
- **Category**: 대분류 카테고리
- **SubCategory**: 소분류 카테고리
- **MenuItem**: 메뉴 아이템
- **Order**: 주문
- **OrderItem**: 주문 항목
- **OrderHistory**: 과거 주문 이력
