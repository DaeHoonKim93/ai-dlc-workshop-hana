# 테이블오더 서비스 - 사용자 스토리

## 분류 방식: User Journey-Based
## 수용 기준: 상세 수준 (Given/When/Then, 정상/예외/경계 케이스)
## 우선순위: 비즈니스 가치 기반 (High/Medium/Low)

# Journey 1: 고객 주문 여정

## Epic 1: 테이블 태블릿 자동 로그인 (FR-01)

### US-01: 태블릿 초기 설정
- **As a** 매니저
- **I want to** 태블릿에 매장 식별자, 테이블 번호, 비밀번호를 1회 설정할 수 있다
- **So that** 고객이 별도 로그인 없이 즉시 주문할 수 있다
- **Priority**: High
- **Persona**: 매니저

**Acceptance Criteria**:
1. **Given** 매니저가 태블릿 초기 설정 화면에 접근했을 때 **When** 매장 식별자, 테이블 번호, 비밀번호를 입력하고 저장 버튼을 누르면 **Then** 설정 정보가 로컬 스토리지에 저장된다
2. **Given** 매니저가 초기 설정을 진행할 때 **When** 필수 필드(매장 식별자, 테이블 번호, 비밀번호)를 비워두면 **Then** 유효성 검증 에러 메시지가 표시된다
3. **Given** 매니저가 유효한 설정 정보를 입력했을 때 **When** 서버 인증이 성공하면 **Then** 자동 로그인이 활성화되고 메뉴 화면으로 이동한다
4. **Given** 매니저가 설정 정보를 입력했을 때 **When** 서버 인증이 실패하면 **Then** 에러 메시지가 표시되고 설정 화면이 유지된다
5. **Given** 매니저가 존재하지 않는 매장 식별자를 입력했을 때 **When** 저장을 시도하면 **Then** "매장을 찾을 수 없습니다" 에러가 표시된다

### US-02: 태블릿 자동 로그인
- **As a** 고객
- **I want to** 태블릿을 켜면 자동으로 로그인되어 메뉴 화면이 표시된다
- **So that** 별도 로그인 절차 없이 즉시 주문할 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 태블릿에 초기 설정이 완료되어 있을 때 **When** 태블릿 브라우저를 열면 **Then** 저장된 정보로 자동 로그인되어 메뉴 화면이 표시된다
2. **Given** 자동 로그인 세션이 유효할 때 **When** 페이지를 새로고침하면 **Then** 세션이 유지되어 메뉴 화면이 다시 표시된다
3. **Given** 자동 로그인 세션이 만료(16시간)되었을 때 **When** 태블릿에 접근하면 **Then** 저장된 정보로 자동 재로그인을 시도한다
4. **Given** 로컬 스토리지에 설정 정보가 없을 때 **When** 태블릿 브라우저를 열면 **Then** 초기 설정 화면이 표시된다
5. **Given** 자동 재로그인이 실패했을 때 **When** 서버 연결 문제가 발생하면 **Then** 에러 메시지와 재시도 버튼이 표시된다

## Epic 2: 메뉴 조회 및 탐색 (FR-02)

### US-03: 카테고리별 메뉴 탐색
- **As a** 고객
- **I want to** 대분류와 소분류 카테고리로 메뉴를 탐색할 수 있다
- **So that** 원하는 메뉴를 빠르게 찾을 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 메뉴 화면에 접근했을 때 **When** 화면이 로드되면 **Then** 대분류 카테고리 목록이 표시된다
2. **Given** 대분류 카테고리가 표시될 때 **When** 특정 대분류를 선택하면 **Then** 해당 대분류의 소분류 카테고리 목록이 표시된다
3. **Given** 소분류 카테고리가 표시될 때 **When** 특정 소분류를 선택하면 **Then** 해당 소분류에 속한 메뉴 아이템 목록이 카드 형태로 표시된다
4. **Given** 메뉴 목록이 표시될 때 **When** 다른 카테고리를 선택하면 **Then** 해당 카테고리의 메뉴로 즉시 전환된다
5. **Given** 특정 카테고리에 메뉴가 없을 때 **When** 해당 카테고리를 선택하면 **Then** "등록된 메뉴가 없습니다" 메시지가 표시된다

### US-04: 메뉴 상세 정보 확인
- **As a** 고객
- **I want to** 메뉴의 상세 정보(이름, 가격, 설명, 이미지)를 확인할 수 있다
- **So that** 주문 전에 메뉴에 대해 충분히 파악할 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 메뉴 카드가 표시될 때 **When** 카드를 확인하면 **Then** 메뉴명, 가격, 이미지가 표시된다
2. **Given** 메뉴 카드가 표시될 때 **When** 카드를 터치하면 **Then** 메뉴 상세 정보(설명 포함)가 표시된다
3. **Given** 메뉴 이미지가 로드 실패했을 때 **When** 카드가 표시되면 **Then** 기본 플레이스홀더 이미지가 표시된다
4. **Given** 메뉴 상세 화면이 열려 있을 때 **When** 닫기 버튼을 누르면 **Then** 메뉴 목록 화면으로 돌아간다
5. **Given** 메뉴 카드의 버튼 크기가 **When** 렌더링될 때 **Then** 최소 44x44px 이상의 터치 타겟을 가진다

## Epic 3: 장바구니 관리 (FR-03)

### US-05: 메뉴를 장바구니에 추가
- **As a** 고객
- **I want to** 원하는 메뉴를 장바구니에 추가할 수 있다
- **So that** 여러 메뉴를 모아서 한 번에 주문할 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 메뉴 카드가 표시될 때 **When** 장바구니 추가 버튼을 누르면 **Then** 해당 메뉴가 수량 1로 장바구니에 추가된다
2. **Given** 이미 장바구니에 있는 메뉴를 **When** 다시 추가하면 **Then** 해당 메뉴의 수량이 1 증가한다
3. **Given** 메뉴가 장바구니에 추가되었을 때 **When** 장바구니 아이콘을 확인하면 **Then** 장바구니 내 총 아이템 수가 표시된다
4. **Given** 장바구니에 메뉴가 추가되었을 때 **When** 페이지를 새로고침하면 **Then** 장바구니 내용이 유지된다 (로컬 스토리지)
5. **Given** 장바구니 추가 버튼을 누를 때 **When** 추가가 성공하면 **Then** 시각적 피드백(토스트 메시지 등)이 표시된다

---

### US-06: 장바구니 수량 조절
- **As a** 고객
- **I want to** 장바구니에 담긴 메뉴의 수량을 조절할 수 있다
- **So that** 원하는 만큼 정확하게 주문할 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 장바구니에 메뉴가 있을 때 **When** 수량 증가(+) 버튼을 누르면 **Then** 해당 메뉴의 수량이 1 증가한다
2. **Given** 장바구니에 수량이 2 이상인 메뉴가 있을 때 **When** 수량 감소(-) 버튼을 누르면 **Then** 해당 메뉴의 수량이 1 감소한다
3. **Given** 장바구니에 수량이 1인 메뉴가 있을 때 **When** 수량 감소(-) 버튼을 누르면 **Then** 해당 메뉴가 장바구니에서 삭제된다
4. **Given** 수량이 변경될 때 **When** 총 금액을 확인하면 **Then** 실시간으로 재계산된 총 금액이 표시된다
5. **Given** 수량이 변경될 때 **When** 변경이 완료되면 **Then** 로컬 스토리지에 즉시 반영된다

---

### US-07: 장바구니 비우기 및 삭제
- **As a** 고객
- **I want to** 장바구니를 비우거나 특정 메뉴를 삭제할 수 있다
- **So that** 주문 내용을 자유롭게 수정할 수 있다
- **Priority**: Medium
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 장바구니에 메뉴가 있을 때 **When** 특정 메뉴의 삭제 버튼을 누르면 **Then** 해당 메뉴가 장바구니에서 제거된다
2. **Given** 장바구니에 메뉴가 있을 때 **When** 장바구니 비우기 버튼을 누르면 **Then** 확인 팝업이 표시된다
3. **Given** 장바구니 비우기 확인 팝업이 표시될 때 **When** 확인을 누르면 **Then** 장바구니의 모든 메뉴가 삭제되고 총 금액이 0원이 된다
4. **Given** 장바구니 비우기 확인 팝업이 표시될 때 **When** 취소를 누르면 **Then** 장바구니 내용이 유지된다
5. **Given** 장바구니가 비어있을 때 **When** 장바구니 화면을 열면 **Then** "장바구니가 비어있습니다" 메시지와 메뉴 보기 버튼이 표시된다

## Epic 4: 주문 생성 (FR-04)

### US-08: 주문 내역 최종 확인
- **As a** 고객
- **I want to** 주문 전에 장바구니 내역을 최종 확인할 수 있다
- **So that** 실수 없이 정확한 주문을 할 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 장바구니에 메뉴가 있을 때 **When** 주문하기 버튼을 누르면 **Then** 주문 확인 화면이 표시된다
2. **Given** 주문 확인 화면이 표시될 때 **When** 내역을 확인하면 **Then** 각 메뉴명, 수량, 단가, 소계 및 총 금액이 표시된다
3. **Given** 주문 확인 화면에서 **When** 뒤로가기를 누르면 **Then** 장바구니 화면으로 돌아가 수정할 수 있다
4. **Given** 장바구니가 비어있을 때 **When** 주문하기 버튼을 누르면 **Then** "장바구니에 메뉴를 추가해주세요" 메시지가 표시된다
5. **Given** 주문 확인 화면이 표시될 때 **When** 테이블 정보를 확인하면 **Then** 현재 테이블 번호가 표시된다

---

### US-09: 주문 확정 및 성공 처리
- **As a** 고객
- **I want to** 주문을 확정하고 성공 결과를 확인할 수 있다
- **So that** 주문이 정상적으로 접수되었음을 알 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 주문 확인 화면에서 **When** 주문 확정 버튼을 누르면 **Then** 서버에 주문이 전송된다
2. **Given** 주문이 성공적으로 전송되었을 때 **When** 서버 응답을 받으면 **Then** 주문 번호가 5초간 표시된다
3. **Given** 주문 성공 화면이 5초간 표시된 후 **When** 타이머가 완료되면 **Then** 장바구니가 자동으로 비워지고 메뉴 화면으로 리다이렉트된다
4. **Given** 주문 전송 중일 때 **When** 서버 응답을 기다리는 동안 **Then** 로딩 인디케이터가 표시되고 중복 주문 방지를 위해 버튼이 비활성화된다
5. **Given** 주문 정보가 서버에 전송될 때 **When** 데이터를 확인하면 **Then** 매장 식별 정보, 테이블 식별 정보, 주문 메뉴 목록, 총 금액, 세션 ID가 포함된다

---

### US-10: 주문 실패 처리
- **As a** 고객
- **I want to** 주문 실패 시 적절한 안내를 받을 수 있다
- **So that** 문제를 인지하고 재시도할 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 주문 전송이 실패했을 때 **When** 서버 에러 응답을 받으면 **Then** 사용자 친화적인 에러 메시지가 표시된다
2. **Given** 주문이 실패했을 때 **When** 에러 메시지가 표시되면 **Then** 장바구니 내용이 유지된다
3. **Given** 네트워크 연결이 끊어졌을 때 **When** 주문을 시도하면 **Then** "네트워크 연결을 확인해주세요" 메시지가 표시된다
4. **Given** 주문 실패 후 **When** 재시도 버튼을 누르면 **Then** 동일한 주문이 다시 전송된다
5. **Given** 서버가 타임아웃되었을 때 **When** 응답이 없으면 **Then** 적절한 타임아웃 메시지가 표시된다

## Epic 5: 주문 내역 조회 (FR-05)

### US-11: 현재 세션 주문 내역 조회
- **As a** 고객
- **I want to** 현재 테이블 세션의 주문 내역을 확인할 수 있다
- **So that** 지금까지 주문한 내용과 상태를 파악할 수 있다
- **Priority**: High
- **Persona**: 고객

**Acceptance Criteria**:
1. **Given** 주문 내역 탭을 선택했을 때 **When** 화면이 로드되면 **Then** 현재 테이블 세션의 주문 목록이 시간 순으로 표시된다
2. **Given** 주문 목록이 표시될 때 **When** 각 주문을 확인하면 **Then** 주문 번호, 주문 시각, 주문 메뉴 및 수량, 주문 금액, 주문 상태가 표시된다
3. **Given** 주문 상태가 변경되었을 때 **When** 목록을 확인하면 **Then** 최신 상태(대기중/접수/준비중/완료)가 반영되어 표시된다
4. **Given** 이전 세션의 주문이 있을 때 **When** 주문 내역을 조회하면 **Then** 이전 세션 주문은 표시되지 않는다
5. **Given** 현재 세션에 주문이 없을 때 **When** 주문 내역을 조회하면 **Then** "주문 내역이 없습니다" 메시지가 표시된다
6. **Given** 주문 내역이 많을 때 **When** 스크롤하면 **Then** 페이지네이션 또는 무한 스크롤로 추가 주문이 로드된다

# Journey 2: 관리자 운영 여정

## Epic 6: 관리자 인증 (FR-06)

### US-12: 관리자 로그인
- **As a** 매니저/직원
- **I want to** 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있다
- **So that** 매장 관리 시스템에 접근할 수 있다
- **Priority**: High
- **Persona**: 매니저, 직원

**Acceptance Criteria**:
1. **Given** 관리자 로그인 페이지에 접근했을 때 **When** 유효한 매장 식별자, 사용자명, 비밀번호를 입력하면 **Then** 로그인이 성공하고 대시보드로 이동한다
2. **Given** 로그인 시도 시 **When** 잘못된 비밀번호를 입력하면 **Then** "인증 정보가 올바르지 않습니다" 에러가 표시된다
3. **Given** 로그인이 성공했을 때 **When** JWT 토큰을 확인하면 **Then** 사용자 역할(매니저/직원) 정보가 포함되어 있다

4. **Given** 수정 폼에서 **When** 취소 버튼을 누르면 **Then** 변경 사항이 저장되지 않고 목록으로 돌아간다
5. **Given** 수정 시 **When** 필수 필드를 비우면 **Then** 유효성 검증 에러가 표시된다

### US-24: 메뉴 삭제
- **As a** 매니저
- **I want to** 더 이상 판매하지 않는 메뉴를 삭제할 수 있다
- **So that** 고객에게 정확한 메뉴만 표시할 수 있다
- **Priority**: Medium
- **Persona**: 매니저 (전용)

**Acceptance Criteria**:
1. **Given** 메뉴 목록에서 **When** 특정 메뉴의 삭제 버튼을 누르면 **Then** 확인 팝업이 표시된다
2. **Given** 삭제 확인 팝업에서 **When** 확인을 누르면 **Then** 메뉴가 삭제된다
3. **Given** 삭제 확인 팝업에서 **When** 취소를 누르면 **Then** 메뉴가 유지된다
4. **Given** 메뉴가 삭제되었을 때 **When** 고객 태블릿에서 메뉴를 조회하면 **Then** 삭제된 메뉴가 표시되지 않는다
5. **Given** 메뉴 삭제가 실패했을 때 **When** 에러가 발생하면 **Then** 에러 메시지가 표시되고 메뉴가 유지된다

---

### US-25: 메뉴 노출 순서 조정
- **As a** 매니저
- **I want to** 메뉴의 노출 순서를 조정할 수 있다
- **So that** 인기 메뉴나 추천 메뉴를 상단에 배치할 수 있다
- **Priority**: Low
- **Persona**: 매니저 (전용)

**Acceptance Criteria**:
1. **Given** 메뉴 관리 화면에서 **When** 메뉴 순서 변경 모드를 활성화하면 **Then** 드래그 앤 드롭으로 순서를 변경할 수 있다
2. **Given** 순서가 변경되었을 때 **When** 저장 버튼을 누르면 **Then** 변경된 순서가 서버에 저장된다
3. **Given** 순서가 저장되었을 때 **When** 고객 태블릿에서 메뉴를 조회하면 **Then** 변경된 순서로 메뉴가 표시된다
4. **Given** 순서 변경 중 **When** 취소 버튼을 누르면 **Then** 원래 순서로 복원된다
5. **Given** 카테고리 내에서 **When** 순서를 변경하면 **Then** 해당 카테고리 내에서만 순서가 변경된다

### US-26: 메뉴 조회 (관리자)
- **As a** 매니저
- **I want to** 카테고리별로 등록된 메뉴를 조회할 수 있다
- **So that** 현재 메뉴 현황을 파악하고 관리할 수 있다
- **Priority**: High
- **Persona**: 매니저 (전용)

**Acceptance Criteria**:
1. **Given** 메뉴 관리 화면에 접근했을 때 **When** 화면이 로드되면 **Then** 카테고리별로 메뉴 목록이 표시된다
2. **Given** 메뉴 목록이 표시될 때 **When** 특정 카테고리를 선택하면 **Then** 해당 카테고리의 메뉴만 필터링되어 표시된다
3. **Given** 메뉴 목록에서 **When** 각 메뉴를 확인하면 **Then** 메뉴명, 가격, 카테고리, 이미지, 등록일이 표시된다
4. **Given** 등록된 메뉴가 없을 때 **When** 메뉴 관리 화면을 열면 **Then** "등록된 메뉴가 없습니다" 메시지와 메뉴 추가 버튼이 표시된다
5. **Given** 직원으로 로그인했을 때 **When** 메뉴 관리 화면에 접근하면 **Then** 접근이 거부된다
